# 核心发射及供弹模块 (LauncherModule) 指南

**对应代码文件：** `src/LauncherModule.cpp`

这是整套系统**执行力**的核心枢纽，用于驱动 **Adafruit Motor Shield V2 (I2C驱动板)** 来高精度（使用外部硬件中断测速）压榨两颗 N20 直流减速双头带编码器电机的算力。因为我们需要 N20 电机把那个拉杆拉到**绝不多一分、绝不少一毫**恰好能卡进下一次激发位的地步。

## 1. 代码解读
- **外部中断函数 (`ISR_FeedEncoderA` 和 `ISR_LaunchEncoderA`)**：由于我们需要电机跑得飞快不丢步，这两行微小的函数直接被挂在了硬件级的底层中断里：只要磁铁扫过了霍尔针脚产生了一次电平跳变（哪怕 `main` 函数里的代码卡死了），计数器变量 `volatile long count` 也一定会自增 1。
- **单向旋转等待法 (`while(xxx < 目标) {}`)**：当主程序发出开火调用的那一刻，系统会全力向电机加电压并直接进入死循环阻塞。直到电机的屁股旋转超过了物理学计算好的圈数后，立刻通过 `run(RELEASE)` 断电抱死，完成一个漂亮的齿轮闭环单程运作。

## 2. 调试与调整指南
组装上传该程序后进入 **自动比赛打靶模式**。如果在发射和供弹过程遇到了位移上的严重偏差，你需要调整下列宏定义（文件上方 `11` - `18` 行附近）：

| 故障现象与调试目标 | 需要修改的地方 | 原因与详细说明 |
| :--- | :--- | :--- |
| **发射拉杆不往前推，在往回走** | `launchMotor->run(FORWARD)` 改为 `BACKWARD` (第 `51` / `65` 行代码附近)| 如果接 N20 电机的正负极线插反了，它自然会反着转导致拉不了皮筋。要么拔线反插要么改代码。供弹一样 `feedMotor->run(...)`。 |
| **转速太慢(没力气) / 转速太快(冲过头停不住)** | `#define LAUNCH_SPEED 255`<br>`#define FEED_SPEED 150` | `255` 是输出给拓展板的满功率。如果冲过头导致的机械受损可以把 `255` 降为 `220`；如果供弹推不动可以把 `150` 提高到 `200` 或 `255`。 |
| **【关键定位】发射机转的不是一圈，而是比如一圈半或半圈就停了** | `#define LAUNCH_PULSES_PER_REV 2086` | 这个数字是靠 `298减速比 * 7线数霍尔` 算出来的。如果你的 N20 转不够或者转多了，多半是商家的参数不准/你买的是 11 线的霍尔针。假如它实际只转了半圈，请试着直接把 `2086` 翻倍改成 `4172` 等直到它正正好转一圈。 |
| **【关键定位】供弹一次并不是标准的 60 度，下巴没对齐** | `#define FEED_PULSES_PER_SLOT 347` | 该值为 `2086 / 6`（6个档位）。如果每次都少推了一点导致卡壳，可以通过不断微调这个数字（比如改为 `350` 然后用串口打印一直去测它是否能恰巧落在凹槽内）。 |
