# 主程序 (Main) 使用指南

**对应代码文件：** `src/main.cpp`

这是程序的“交通指挥塔”。你在这里不会看到具体的电机是怎么转的，也看不到 EEPROM 是怎么被写入的；它唯一的职责就是：**读取拨动开关 -> 梳理大业务逻辑 -> 下放指令给各个子模块。**

## 1. 代码解读
- **`setup()` 环境初始化**：调用子模块的各自的 `.Init()` 函数，最后负责从 EEPROM 读取已记忆的坐标清单，以便设备一旦断电重启，接通还是能直接拿着原数据去比赛。
- **`loop()` 系统主循环**：
  1. 每循环一次都去检查 `PIN_MODE_SWITCH` 所连接的那个拨动开关，判断是应该处于比赛还是调试模式。
  2. 如果由于操作发生了**模式切换**（比如刚调试完毕，拨动开关至比赛模式），系统会将 `currentTargetIndex` 置 0 (即从第一个靶子重新开始打)。
  3. **分流逻辑执行**：
     - 若在 `MODE_TUNING`：单纯地无限循环调用 `Gimbal_RunTuningMode()` 去听摇杆的话；
     - 若在 `MODE_COMPETITION`：则按照设定的串行流程执行：读取 EEPROM 中的一个目标 -> 操控云台移动并等待它到达 -> 射击发射 -> 复位装回弹药 -> 间隔短时间进入下一个目标的循环。直至所有坐标执行完毕进入发呆状态（避免误触）。

## 2. 调试与调整指南
若你想调整整个**连招系统的节奏**（比如有些时候云台转过去了它还在晃，发射太快了导致打不准），请在这里修改。

| 调整目标 | 需要修改的地方 | 说明 |
| :--- | :--- | :--- |
| **云台瞄准后的稳定期** | `delay(500); // 等待云台完全稳定` | 当云台猛烈转向目标后，因机械惯性可能需要等待静止。如果你发现刚扭头瞬间子弹就打出去了，把这个数值改大（比如 `1000` 即等待 1 秒）。 |
| **发射与供弹之间的间隔** | `delay(200);` (位于 `Launcher_Fire` 之后) | 控制连贯性。确保发射完全结束后再去上下一发子弹。 |
| **打完所有的靶子后希望它重新循环** | `else { }` (第 64 行) | 目前在所有目标打完后进入空跑等待状态避免浪费子弹。如果想要它打完后重新开始扫射一圈，可以在这块加上 `currentTargetIndex = 0;`。 |
| **拨动开关防反** | `bool isCompetition = (digitalRead(PIN_MODE_SWITCH) == LOW);` | 如果你发现开关往上拨进入了比赛，往下拨却变成了调试，和你的习惯反了，这里把 `LOW` 改为 `HIGH` 即可。 |
