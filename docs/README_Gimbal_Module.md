# 云台控制与存储模块 (GimbalModule) 指南

**对应代码文件：** `src/GimbalModule.cpp`

该模块专门负责 **[摇杆 -> Yaw/Pitch舵机]** 的人工跟随时序计算，以及 **断电坐标保存(EEPROM)**。

## 1. 代码解读
- **开关量（WASD）纯数字改造**：由于比赛规则不再允许使用模拟量，摇杆识别已剔除死区过滤逻辑，改为纯粹的 `digitalRead()` 进行“接通即走、松开即停”的恒速移动逻辑。四个方向的开关按钮只需一端接地 (GND)，另一端接入控制引脚即可，由单片机提供内部上拉。
- **EEPROM 保存机制**：单片机具有类似电脑硬盘特性的闪存块（哪怕断电也是持久化保留数据），我们通过 `EEPROM.put()` 直接将带有着 Yaw 和 Pitch 两个分量的 C++ 结构体，原封不动地写入硬件的内存里。
- **循环复写保护**：当你按下了第六次按键（也就是 `savedTargetCount >= MAX_TARGETS`），系统不仅会把坐标数量清零，甚至连存放在 0 号地址的标记位也会被改写，让你随时能重新标定无需重新烧录程序。

## 2. 调试与调整指南

当你连接好舵机并进入 **“调试模式”** 测试时，你可能需要根据实际手感调整以下参数（均在 `src/GimbalModule.cpp` 中）：

| 调整目标 | 需要修改的参数 | 说明 |
| :--- | :--- | :--- |
| **云台上下/左右转反了** | 替换 `+= 1` 和 `-= 1` | 拨动按键时，如果觉得往上应该是抬头，反而低头了。去找到 `if (isUp) currentPitch += 1;` 并把它改成 `-= 1`。同样适用于四向中的其他方向。 |
| **转动太快/太慢** | `delay(15);` (靠近函数末尾) | 舵机每次更新 1 度之后的等待时间。如果觉得一直按着方向键转得太快导致难以精准瞄准靶心，把延时改成 `delay(25)` 或者 `30`，云台就会降速。 |
| **按键总是不灵或一次性录入了好多坐标**| `const unsigned long debounceDelay = 50;` | 按键的物理防抖掩码，按一次记录多次（抽风）的话，可以将防抖调大至 `80` 到 `100`。 |
