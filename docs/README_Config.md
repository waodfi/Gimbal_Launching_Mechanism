# Config 配置文件指南

**对应代码文件：** `include/Config.h`

`Config.h` 是整个项目的大脑地图。所有的硬件设备究竟插在 Arduino 的哪一个引脚上，以及系统中多个模块需要共享的特殊数据结构（比如状态枚举、坐标结构体）都在这里定义。

## 1. 代码解读
- **硬件引脚分配区 (`#define PIN_...`)**：定义了开关、摇杆、舵机和编码器的物理接线位置。这能保证我们在改代码的时候，不需要满篇找 `digitalRead(4)`，而是直接阅读 `digitalRead(PIN_MODE_SWITCH)`，极大提升代码可读性。
- **系统核心状态枚举 (`SystemMode`)**：系统被划分为 `MODE_TUNING`（调试模式）和 `MODE_COMPETITION`（比赛模式）。
- **结构体 (`TargetPosition`)**：把 Yaw 和 Pitch 两个分离的坐标绑定在了一起，专门用于生成坐标数组，方便存入 EEPROM 中。

## 2. 调试与调整指南
如果你的接线发生了变化，或者你想增加坐标记录数量，请在这个文件中修改：

| 调整目标 | 需要修改的参数 | 说明 |
| :--- | :--- | :--- |
| **修改摇杆/按键/开关的引脚** | `PIN_JOY_X`, `PIN_JOY_Y`, `PIN_JOY_BTN`, `PIN_MODE_SWITCH` | 只要确保模拟信号(X/Y)插在 `A` 开头的模拟口，数字按键插在 `D` 数字口即可。 |
| **修改舵机引脚** | `PIN_SERVO_YAW`, `PIN_SERVO_PITCH` | 必须使用带有 PWM 功能（带 `~` 号）的引脚，如 `9`, `10`, `11` 等。 |
| **更换 N20 电机编码器引脚** | `PIN_FEED_ENCA`, `PIN_LAUNCH_ENCA` | **【极其重要：不可随意修改】**！对于 Arduino Uno 硬件外部中断只有两个：`D2` (中断0) 和 `D3` (中断1)。两个电机的 A 相**必须并只能**接在这里才能保证编码器高频计数不丢步。B相（如 `PIN_LAUNCH_ENCB` 等）你可以插在任何其他普通引脚上。 |
| **修改最大记录目标数** | `#define MAX_TARGETS 5` | 如果你希望系统在调试模式能存 10 个打靶点，就把 5 改成 10（注意这会成倍消耗 EEPROM 存储空间，但是 Uno 的 1KB 容量足够存几十个没问题）。 |
